# B. Gilman & Co Financial Dashboard

Dynamic Next.js financial dashboard deployed on Vercel.

## Local Development

```bash
pnpm install
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000).

In the Codex desktop environment, `pnpm dev` prepends the bundled local Node runtime before starting Next.js. This avoids a macOS code-signing failure when Next loads its SWC compiler from `node_modules`.

This repository enforces a 14-day package maturity window for JavaScript installs. `npm` reads `min-release-age=14` from `.npmrc`, and `pnpm` reads `minimumReleaseAge: 20160` from `pnpm-workspace.yaml`, so newly published registry versions must age for 14 days before they can be installed.

## Supabase Authentication

Create a Supabase project and copy `.env.example` to `.env.local` with your public Supabase URL, anon key, and server-only service role key. The login and signup screens accept any email address and passwords with at least 4 characters. New accounts must verify their email address before signing in.

This app invites new users through a server route backed by the service role key so Supabase sends the verification email, then stores the submitted password on the invited user. In the Supabase dashboard, keep `Authentication -> Providers -> Email` enabled and keep email confirmations enabled for this project. If you use a hosted production domain, add its `/auth/callback` URL to the Auth redirect allow list alongside your local URLs. Never expose `SUPABASE_SERVICE_ROLE_KEY` to client code.

If you see `Invalid API key`, the most common cause is that `NEXT_PUBLIC_SUPABASE_URL` points to one Supabase project while `NEXT_PUBLIC_SUPABASE_ANON_KEY` or `SUPABASE_SERVICE_ROLE_KEY` came from another. These values must all belong to the same project ref.

Vercel should have real values for `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`; the browser client also includes the current public Supabase anon key as a fallback so login keeps working if Vercel is accidentally left on a placeholder value. Signup and verification email delivery still require `SUPABASE_SERVICE_ROLE_KEY` at runtime.

Supabase may return `email rate limit exceeded` when too many verification or magic-link emails are requested through the default sender. For production, configure a dedicated SMTP provider in Supabase Auth and adjust email rate limits to match expected signup volume.

## Vercel Deployment

Vercel is the only deployment target for this app. Use the default Next.js framework preset with `pnpm install --frozen-lockfile` and `pnpm build`. The app uses dynamic route handlers under `app/api/` for market data, price history, S&P 500 universe data, SEC filings, and authentication, so it should be deployed as a normal Vercel Next.js app rather than a static export or GitHub Pages site.

Market data can come from Vercel environment variables or from the packaged fallback snapshots:

- `MARKET_SNAPSHOT_URL` points to a JSON file with the same shape as `public/data/yfinance-snapshot.json`.
- `SEC_SNAPSHOT_URL` points to a JSON file with the same shape as `public/data/sec-snapshot.json`.

If those variables are not configured, the Vercel API routes fall back to the JSON files in `public/data/`. The S&P 500 universe and SEC filings can also be refreshed server-side at request time when snapshot data is unavailable.

## Automatic Market Data Refresh

The dashboard asks the market API for fresh quotes every 55 seconds. The API attempts live Yahoo Finance quotes on each request and falls back to the packaged yfinance snapshot for any missing or unavailable ticker data.

The `Refresh market data` GitHub Actions workflow updates `public/data/yfinance-snapshot.json` on `main` every 5 minutes during regular U.S. market hours and can also be run manually from GitHub Actions. The snapshot job first upgrades to `pip==26.0.1`, then installs `yfinance` with `PIP_UPLOADED_PRIOR_TO=P14D` so PyPI packages must be at least 14 days old before installation. It then uses 1-minute yfinance bars for quote fields and daily bars for six-month price history, then commits the refreshed snapshot when prices have changed. Vercel will redeploy from the `main` commit so the API routes have a recent packaged fallback without a manual local run.

The Researcher tab also includes a Netflix real-options valuation generated by `scripts/build_netflix_valuation.py` from the local Netflix workbook and research notes, then served from `public/data/netflix-valuation.json`.
