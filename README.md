# B. Gilman & Co Financial Dashboard

Dynamic Next.js financial dashboard deployed on Vercel.

## Local Development

```bash
pnpm install
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000).

## Supabase Authentication

Create a Supabase project and copy `.env.example` to `.env.local` with your public Supabase URL, anon key, and server-only service role key. The login and signup screens accept any email address and passwords with at least 4 characters. New accounts must verify their email address before signing in.

Hosted Supabase Auth requires its project-level minimum password length to be at least 6, so this app creates new users through a server route backed by the service role key and then signs them in with the normal browser Supabase client. Never expose `SUPABASE_SERVICE_ROLE_KEY` to client code.

## Vercel Deployment

Vercel should use the default Next.js framework preset with `pnpm install --frozen-lockfile` and `pnpm build`. The app uses dynamic route handlers under `app/api/` for market data, price history, S&P 500 universe data, and SEC filings, so it should be deployed as a normal Vercel Next.js app rather than a static export.

Market data can come from Vercel environment variables or from the packaged fallback snapshots:

- `MARKET_SNAPSHOT_URL` points to a JSON file with the same shape as `public/data/yfinance-snapshot.json`.
- `SEC_SNAPSHOT_URL` points to a JSON file with the same shape as `public/data/sec-snapshot.json`.

If those variables are not configured, the Vercel API routes fall back to the JSON files in `public/data/`. The S&P 500 universe and SEC filings can also be refreshed server-side at request time when snapshot data is unavailable.

The Researcher tab also includes a Netflix real-options valuation generated by `scripts/build_netflix_valuation.py` from the local Netflix workbook and research notes, then served from `public/data/netflix-valuation.json`.
